Take-Home Assessment – Software Testing Tool Evaluation

Weight: 10% · Group size: 2–3 students · Time per group: 6 minutes (3 min presentation + 3 min live demo)

SE3112 · Testing No Selenium or Postman Live Demo Required
Due date: 28 October 2025
Presentation schedule: Will be posted later on Moodle.

Objective

Evaluate a software testing tool other than Selenium or Postman. Research the tool and deliver a short presentation plus a live demo using actual working code.

Tool Selection

Choose one tool (examples include, but are not limited to): JUnit, PyTest, TestNG, Cucumber, Cypress, Playwright, Robot Framework, Karate, JMeter, Gatling, nUnit, Mocha/Chai, Vitest, RSpec, Behave, K6, Locust. Do not use Selenium or Postman.

Deliverables

  1. Presentation (3 minutes) — Cover: Introduction (purpose & domain), Core features, How it is used (workflow, typical setup), How widely used (adoption evidence: ecosystem, community, notable users).
  2. Live Demo (3 minutes) — Show quick setup & run a small but working code example that demonstrates at least one key feature (e.g., assertions, fixtures, mocking, BDD syntax, reporting, CI usage).
  3. Submission package — Upload to Moodle before your slot:
    • Slides (max 5) or a 1-page summary (PDF)
    • Demo source code + minimal setup instructions (README)
  4. Participation — Each student in the group must present part of the work (either in the 3-min talk or the 3-min demo).

Guidelines & Tips

  • Limit slides to the essentials; use visuals (architecture, workflow) over dense text.
  • Rehearse the demo flow; keep commands/code ready. Have a short fallback video in case of technical issues.
  • Show at least one differentiator (e.g., Playwright’s auto-waiting, PyTest fixtures, Cucumber Gherkin syntax, Gatling’s Scala DSL).
  • Optional: show how the tool plugs into CI (e.g., GitHub Actions) or reports coverage/results.

Academic Integrity

Use your own words and code. Cite any references and clearly attribute borrowed snippets. Be prepared to answer questions about your demo.
